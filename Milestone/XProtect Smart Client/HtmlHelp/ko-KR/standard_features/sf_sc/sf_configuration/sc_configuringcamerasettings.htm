카메라(구성)

이 섹션의 설정은 오버레이 버튼과 속성 창에 적용됩니다. 이 창에 액세스하려면 설정 버튼을 클릭하고 왼편에 탐색 창이 보이는지 확인하십시오.

카메라 설정

속성 창의 설정 모드에서 선택한 카메라의 속성을 보고 편집할 수 있습니다(선택한 카메라는 뷰에서 굵은 테두리로 표시됨).

이름 설명
카메라 이름

선택한 카메라의 이름을 표시합니다.

카메라를 변경하려면 줄임표 버튼을 클릭하여 카메라 선택 창을 열고 다른 카메라를 선택합니다. 카메라를 변경하되 설정을 유지하려는 경우 유용할 수 있습니다.
라이브 스트림

해당하는 경우 뷰에 표시할 라이브 스트림을 선택합니다. 서버에 여러 개의 스트림이 설정되어 있는 경우, 기본값 또는 사용 가능한 스트림 옵션 중에서 하나를 선택할 수 있습니다. 기본값 이외의 다른 옵션을 선택한 경우, 이미지 화질 또는 프레임 속도모션 비디오에 포함된 정보의 양을 나타내는 측정 단위입니다. 일반적으로 FPS(Frames Per second)로 측정됩니다. 설정을 편집할 수 없습니다.
이미지 품질

XProtect Smart Client 에서 표시된 비디오의 이미지 품질을 지정하기는 JPEG 스트리밍을 보고 있는 경우에만 관련있습니다. H264 또는 H265 같은 다른 코덱으로 보고 있으며 전체 보다 낮은 품질로 설정된 경우, JPEG로 재인코딩 시 대역폭, CPU 및 GPU 사용량을 늘리게 됩니다.

비디오가 표시될 때의 비디오 품질을 결정하지만 대역폭 사용량에도 영향을 줍니다. 네트워크 연결 속도가 느린 인터넷에서 XProtect Smart Client 를 사용하거나 다른 이유로 인해 대역폭 사용량을 제한해야 하는 경우 낮음 또는 보통 등을 선택하여 서버 측에서 미지 화질을 낮출 수 있습니다.

낮은 이미지 화질을 선택하면 선택한 카메라의 이미지가 XProtect Smart Client 에 전송되기 전에 감시 시스템 서버에서 JPEG이미지 압축 방법으로서 JPG 또는 Joint Photographic Experts Group이라고도 합니다. 이 방법은 소위 손실 압축 방법으로서 압축하는 동안 일부 이미지 세부사항이 손실됩니다. 이 방법으로 압축된 이미지를 일반적으로 JPG 또는 JPEG라고 합니다. 형식으로 다시 인코딩됩니다. 다음과 같은 옵션에 따라 다시 인코딩됩니다.

최고: 기본 설정으로서 원래 비디오의 최고 품질을 제공합니다.

매우 높음(메가픽셀용): 640픽셀(VGA)의 출력 너비 및 25%의 JPEG 품질 수준으로 다시 인코딩합니다.

높음: 320픽셀(QVGA)의 출력 너비 및 25%의 JPEG 품질 수준으로 다시 인코딩합니다.

보통: 200픽셀의 출력 너비 및 25%의 JPEG 품질 수준으로 다시 인코딩합니다.

낮음: 160픽셀의 출력 너비 및 20%의 JPEG 품질 수준으로 다시 인코딩합니다.

높이의 경우 원래 비디오의 너비 및 가로 세로 비율이미지의 높이/너비 관계입니다.에 따라 크기가 조정됩니다.

이미지 화질 선택 항목은 라이브 비디오 및 녹화 비디오, JPEG 형식 및 MPEG 형식에 적용됩니다. 하지만 MPEG 형식의 경우 라이브 비디오를 볼 때 키프레임만 다시 인코딩되는 반면 녹화 비디오를 볼 때는 모든 프레임이 다시 인코딩됩니다.

낮은 이미지 화질을 사용하면 대역폭 사용을 제한하는 데 도움이 되지만 이미지를 다시 인코딩해야 하므로 감시 시스템 서버에서 추가 리소스를 사용하게 됩니다.

단일 카메라의 이미지 화질을 낮추고 모든 카메라에 적용 버튼을 클릭하여 뷰에 있는 모든 카메라에 대한 대역폭 사용량을 신속하게 줄일 수 있습니다.
최대화 시 유지

라이브 또는 녹화 비디오를 시청하는 경우 특정 카메라 위치를 두 번 클릭해서 최대화할 수 있습니다. 이 때 사용자가 선택한 이미지 화질 옵션과 관계 없이 카메라 비디오가 기본적으로 최고 화질로 표시됩니다.

비디오를 확대했을 때 선택한 이미지 화질도 적용되도록 하려면 이미지 화질 설정 바로 아래 있는 최대화 시 유지 상자를 선택하십시오.
프레임 속도

선택한 카메라의 프레임 속도를 선택합니다. 무제한 (기본값), 보통 또는 낮음 중 하나를 선택합니다. 선택한 프레임 속도와 감시 카메라를 설정하는 방식의 조합(프레임 속도 결과(설명됨) 참조)은 비디오의 품질에 영향을 줍니다.
PTZ 클릭 모드

PTZ 카메라에 대한 기본 PTZ 클릭 모드를 선택합니다. 옵션으로는 가운데로 클릭 또는 가상 조이스틱이 있습니다. 카메라에 대해 다른 기본 PTZ 클릭 모드를 선택하여 개별 카메라에서 이 설정을 재정의할 수 있습니다.
어안렌즈 분할 모드

선택한 카메라가 어안렌즈 카메라인 경우에만 사용할 수 있습니다. 어안렌즈 기술을 사용하면 360° 파노라마 이미지를 만들고 볼 수 있습니다. XProtect Smart Client 은(는) 단일 어안렌즈 카메라에서 최대 4개의 다른 뷰포인트를 지원합니다. 어안렌즈 분할 모드 목록에서 필요한 분할 모드를 선택할 수 있습니다.

분할 안 함 - 단일 뷰포인트를 볼 수 있습니다.

두 개씩 - 한 번에 4개의 다른 뷰포인트를 볼 수 있습니다.

XProtect Smart Client의 탭에서 볼 때, 동일한 이미지의 1개 또는 4개 뷰포인트로 지정된 대로 어안렌즈 카메라가 나타납니다.

어안 카메라의 다른 뷰포인트를 볼 때 각 뷰포인트 내부를 클릭하거나 카메라 도구 모음에서 PTZ 사전 설정 메뉴를 사용하여 각 뷰포인트를 독립적으로 탐색할 수 있습니다.
이미지 가로 세로 비율 유지

이 옵션을 선택하면 비디오를 카메라 위치의 크기에 맞게 늘릴 수 없습니다. 녹화된 가로 세로 비율(높이/너비 관계)로 비디오가 표시됩니다.

이 경우 일부 카메라의 이미지 주변에 가로 또는 세로의 검은색 표시줄이 표시됩니다.

이 선택란을 선택 취소하면 뷰의 해당 위치에 맞게 비디오가 늘려집니다. 이 경우 비디오가 약간 왜곡되지만 비디오 주변에 검은색 표시줄이 나타나지는 않습니다.
모션 시 업데이트

이 옵션을 선택하면 선택한 카메라 비디오는 모션이 감지될 때만 XProtect Smart Client의 라이브 탭에서 업데이트됩니다. 감시 시스템 서버에서 카메라에 대해 구성된 모션 감도에 따라 CPU 사용량을 현저히 줄이는 데 도움이 될 수 있습니다.

모션이 있을 때만 비디오가 업데이트되는 경우 모션이 감지되기 전까지는 뷰의 카메라 위치에서 스틸 이미지와 함께 모션 없음 메시지가 표시됩니다. 스틸 이미지에는 회색 오버레이가 표시되며, 이를 통해 카메라에 모션이 없다는 것을 쉽게 확인할 수 있습니다.
모션 감지 시 경고음

카메라 비디오가 라이브 탭에 표시되면 모션이 감지될 때 간단한 경고음을 낼 수 있습니다.

경고음은 카메라의 비디오가 XProtect Smart Client 에 실제로 표시된 경우에만 작동합니다. 따라서 해당 카메라를 포함하는 창을 최소화한 경우에는 경고음이 작동하지 않습니다. 마찬가지로 한 카메라만 표시되도록 하나의 뷰에 최대화한 경우, 다른 카메라와 관련된 경고음을 들을 수 없습니다.

항상 끄기: 모션이 감지될 때 경고음을 사용하지 않습니다.

항상 켜기: 카메라에서 모션이 감지될 때마다 경고음을 재생합니다.
이벤트감시 시스템에서 발생하는 사전 정의된 인시던트로서 감시 시스템에서 동작을 트리거하는 데 사용됩니다. 감시 시스템 구성에 따라 이벤트는 외부 센서의 입력, 감지된 모션, 다른 응용 프로그램에 의해 발생되거나 사용자 입력을 통해 수동으로 발생시킬 수 있습니다. 예를 들어, 이벤트를 발생시켜 카메라 녹화를 특정 프레임 속도로 만들거나, 출력을 활성화하거나, 전자메일을 보내거나 이들을 조합하는 데 사용할 수 있습니다. 시 경고음

이 기능은 특정 XProtect VMS 시스템에서만 사용할 수 있습니다. 사용 중인 시스템의 기능에 관한 정보는 감시 시스템의 차이점를 참조하십시오.

이 기능을 사용하려면 감시 시스템 서버에 이벤트 알림이 구성되어 있어야 합니다.

경고음은 카메라의 비디오가 XProtect Smart Client 에 실제로 표시된 경우에만 작동합니다. 따라서 해당 카메라를 포함하는 창을 최소화한 경우에는 경고음이 작동하지 않습니다. 마찬가지로 한 카메라만 표시되도록 하나의 뷰에 최대화한 경우, 다른 카메라와 관련된 경고음을 들을 수 없습니다.

카메라 비디오가 라이브 탭에 표시되면 선택한 카메라와 관련된 이벤트가 발생할 때 간단한 경고음을 낼 수 있습니다.

항상 끄기: 카메라와 관련된 이벤트가 발생할 때 경고음을 사용하지 않습니다.

항상 켜기: 카메라와 관련된 이벤트가 발생할 때마다 경고음을 재생합니다.
디스플레이 설정

기본 디스플레이 설정 사용: 제목 표시줄 및 선택된 카메라에 대한 비디오 표시기를 표시하기 위해 설정 창에 정의된 대로 기본 설정을 사용합니다. 선택한 카메라에 기본으로 설정되지 않은 동작이 필요한 경우 선택란을 선택 취소하고 제목 표시줄 및/또는 비디오 표시기의 표시 여부를 선택합니다.

제목 표시줄 표시: 각 카메라 위치 상단에 제목 표시줄을 표시합니다. 제목 표시줄을 통해 카메라를 곧바로 확인할 수 있습니다. 라이브 탭에 표시된 경우 제목 표시줄에는 감지된 모션 및 이벤트, 카메라가 녹화 중인지 여부 등에 관한 정보가 표시됩니다. 카메라 표시기(설명됨)를 참조하십시오.

제목 표시줄이 표시되지 않도록 선택하면 모션과 이벤트에 대한 시각적 표시기를 볼 수 없습니다. 대신 경고음을 사용할 수 있습니다.

바운딩 박스 계층 표시: 개별 카메라에 묶음 상자를 표시합니다. 묶음 상자 제공자(묶음 상자 제공자(설명됨) 참조) 대화 상자를 열고 카메라에 데이터를 제공하도록 메타데이터 장치를 지정합니다.
비디오 버퍼링

속성 창에서 이 부분이 표시되지 않을 수 있습니다. 이를 보려면 설정 창의(XProtect Smart Client 에서의 설정 참조) 기능 탭으로 이동하여, 설정 > 비디오 버퍼링 편집 이 사용 가능 으로 설정되었는지 확인합니다.

지터 없이 부드러운 라이브 비디오 디스플레이가 필요한 경우 비디오 버퍼를 구축할 수 있습니다.

가능한 경우 비디오 버퍼링을 사용하지 마십시오. 비디오 버퍼링은 뷰에 표시되는 각 카메라의 메모리 사용량을 크게 늘릴 수 있습니다. 비디오 버퍼링을 사용해야 하는 경우 버퍼링 수준을 가능하면 낮게 유지하십시오.

라이브 비디오가 버퍼에 저장되어 있는 경우 지터 없이 부드럽게 표시되지만 버퍼를 구축하면 라이브 비디오의 디스플레이에 약간 지연이 발생합니다. 대부분의 경우 비디오를 보는 사용자에게는 이러한 지연이 문제가 되지 않습니다. 하지만 PTZ(이동/기울기/줌) 카메라의 경우, 특히 조이스틱을 사용하여 카메라를 작동하는 경우 지연이 매우 선명하게 느껴질 수 있습니다.

비디오 버퍼링의 양을 제어할 수 있으므로 표시되는 라이브 비디오(버퍼링이 필요하므로 약간의 지연 발생) 또는 즉각적인 PTZ 및 조이스틱 작동(버퍼가 필요하지 않지만, 버퍼 부족으로 인해 라이브 비디오에서 약간의 지터 발생)의 우선 순위를 쉽게 지정할 수 있습니다.

비디오 버퍼링을 사용하려면 기본 비디오 버퍼 사용 을 선택한 다음 필요한 버퍼를 없음 에서 최대 2초 로 선택합니다.
모두 적용

모든 카메라에 적용 버튼을 사용하면 선택한 카메라의 카메라 설정을 뷰에 있는 모든 카메라에 신속하게 적용할 수 있습니다.

묶음 상자(설명됨)

묶음 상자는 예를 들어 카메라 이미지의 개체를 감싸는 사각의 테두리입니다. XProtect Smart Client 에서, 묶음 상자는 비디오에서 노란색 테두리로 표시됩니다.

색상은 사용 중인 VMS 시스템이 어떻게 구성되었느냐에 따라 달라집니다.

카메라 속성의 디스플레이 설정 에서 개별 카메라에 대해 묶음 상자를 표시하거나 숨길 수 있습니다.

묶음 상자가 화면에 표시되는 경우, 다음과 같은 상황에도 묶음 상자가 표시됩니다.

  • XProtect 형식으로 비디오를 내보내기하는 경우. 또한 내보내기 설정를 참조하십시오.

  • 스틸 이미지를 출력하는 경우. 또한 감시 보고서 인쇄 또는 생성를 참조하십시오.

묶음 상자 제공자(설명됨)

바운딩 박스 계층 표시 가 선택되어 있어야 합니다. 대화 상자에서 이 카메라의 비디오에 있는 바운딩 박스 데이터를 제공하고자 하는 메타데이터 장치를 활성화합니다. 장치의 목록은 시스템 관리자가 결정합니다.

오버레이 버튼(설명됨)

보조 명령(카메라에 의해 정의된 명령)을 트리거하기 위해 뷰에서 카메라 위치에 오버레이 버튼을 추가할 수 있습니다. 오버레이 버튼은 감시 시스템에 따라 달라질 수 있습니다(감시 시스템의 차이점 참조). 보조 명령어는 카메라마다 다릅니다. 자세한 내용은 카메라의 설명서를 참조하십시오.

경고음(설명됨)

다음과 같은 경우 사용자에게 경고음을 알리도록 XProtect Smart Client 가 구성되었을 수 있습니다.

  • 하나 이상의 특정 카메라에서 모션이 감지되는 경우
  • 하나 이상의 특정 카메라와 관련된 이벤트(이벤트(설명됨) 참조)가 발생하는 경우

경고음이 들리면 주의를 기울여야 합니다. 해당 조직에서 경고음 사용 여부 및 사용 방법에 대한 의문 사항은 시스템 관리자에게 문의하십시오.

특정 카메라의 경고음을 일시적으로 음소거할 수 있습니다. 카메라 도구 모음에서 기타 > 경고음 > 음소거 를 클릭합니다.

XProtect Smart Client 창을 최소화하면 경고음이 비활성화됩니다.

카메라의 경고음을 다시 켜려면 기타 > 경고음 > 음소거를 다시 클릭합니다.
