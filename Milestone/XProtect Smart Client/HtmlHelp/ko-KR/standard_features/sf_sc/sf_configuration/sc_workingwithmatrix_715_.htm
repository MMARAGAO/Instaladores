Matrix (사용량)

이 기능은 특정 XProtect VMS 시스템에서만 사용할 수 있습니다. 사용 중인 시스템의 기능에 관한 정보는 감시 시스템의 차이점를 참조하십시오.

Matrix (설명됨)

Matrix 는 감시 시스템 카메라의 비디오를 네트워크의 모니터(Matrix 수신자라고 함)로 보내거나 받을 수 있는 기능입니다. 일반적인 Matrix 구성에서는 정의된 이벤트가 발생하면(예: 모션이 감지될 때 또는 다른 사용자가 중요한 라이브 비디오를 공유하려고 할 때) 필요한 Matrix 수신자에게 라이브 비디오가 자동으로 표시됩니다. 감시 시스템 서버에 Matrix 이(가) 구성되어 있으면 Matrix 콘텐츠를 사용자의 XProtect Smart Client 뷰에 포함시킬 수 있습니다. 특정 이벤트가 발생하거나 다른 사용자가 비디오를 공유하려는 경우, 라이브 비디오가 사용자의 Matrix 뷰에 자동으로 나타납니다.

Matrix 콘텐츠 보기(설명됨)

MatrixXProtect 감시 시스템의 추가 기능으로 사용할 수 있는 제품입니다.XProtect Incident Manager을(를) 사용하면 XProtect Smart Client내에서 사건을 문서화하고 관리할 수 있습니다. 설정에 사용되는 이벤트감시 시스템에서 발생하는 사전 정의된 인시던트로서 감시 시스템에서 동작을 트리거하는 데 사용됩니다. 감시 시스템 구성에 따라 이벤트는 외부 센서의 입력, 감지된 모션, 다른 응용 프로그램에 의해 발생되거나 사용자 입력을 통해 수동으로 발생시킬 수 있습니다. 예를 들어, 이벤트를 발생시켜 카메라 녹화를 특정 프레임 속도로 만들거나, 출력을 활성화하거나, 전자메일을 보내거나 이들을 조합하는 데 사용할 수 있습니다. 또는 카메라는 감시 시스템 서버의 Matrix 구성 또는 다른 사용자가 해당 사용자와 공유하려는 비디오에 따라 완전히 다릅니다. XProtect Smart Client 에서는 이를 제어할 수 없습니다. 그러나 필요에 따라 뷰의 여러 위치에 Matrix 콘텐츠를 추가할 수 있으므로 Matrix 트리거된 여러 소스의 라이브 비디오를 동시에 볼 수 있습니다.

Matrix 위치는 도구 모음에서 Matrix 아이콘으로 표시됩니다. . Matrix 을(를) 두 번 클릭하여 최대화할 수 있습니다.

하나의 뷰에 여러 Matrix 위치가 포함될 수 있습니다. 이 방법을 통해 여러 Matrix 트리거된 소스의 라이브 비디오를 동시에 볼 수 있습니다. 뷰에 여러 Matrix 위치가 있는 경우 이러한 위치의 순위가 매겨집니다. 이 위치 중 하나가 기본 Matrix 위치가 되고 다른 위치가 보조 위치 등으로 설정됩니다. 첫 번째 Matrix 트리거된 라이브 비디오 스트림이 수신되면 이 스트림이 기본 Matrix 위치에 자동으로 표시됩니다. Matrix 트리거된 다음 비디오 스트림을 수신할 때, FIFO(선입선출법) 원리가 적용됩니다. 이전에 수신된 비디오 스트림이 사용자 뷰의 보조 Matrix 위치에 전송되고, 최신 비디오 스트림이 기본 Matrix 위치에 배치되는 방식입니다. Matrix 위치의 순위가 자동 적용됩니다. 추가하는 첫 번째 Matrix 위치가 기본 Matrix 위치가 되고, 다음으로 추가한 위치가 보조 위치가 되는 식입니다. 설정 모드에서 이 순위를 변경할 수 있습니다.

재생 탭에서 Matrix 위치는 라이브 탭에서 Matrix 위치가 마지막으로 사용된 카메라의 비디오를 표시합니다. 재생 탭의 탐색 기능을 이용해 이 비디오를 재생할 수 있습니다.

비디오를 수동으로 Matrix 수신자에게 보내기

비디오를 핫스팟(핫스팟(설명됨) 참조) 또는 순환보기(순환보기(설명됨) 참조)로 보낼 수 없습니다.

요구사항

Matrix 콘텐츠가 뷰에 추가되었습니다. 뷰에 Matrix 추가를 참조하십시오.

  1. 뷰를 선택합니다.
  2. 카메라 도구 모음에서 기타 > Matrix 를 클릭한 다음 해당 Matrix 수신자를 선택합니다.
