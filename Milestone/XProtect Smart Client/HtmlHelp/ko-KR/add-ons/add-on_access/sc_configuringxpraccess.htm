XProtect Access (구성)

뷰에 액세스 모니터 추가

우선 액세스 제어에 대한 뷰 항목을 정의합니다.

  1. 설정 모드의 라이브 탭 (라이브 탭(설명됨) 참조)에서 액세스 제어 모니터링에 사용하고자 하는 뷰를 선택하십시오.
  2. 시스템 개요 창에서 액세스 모니터를 클릭하여 이를 뷰 항목으로 끌어서 놓습니다.
  3. 화면에 표시되는 액세스 모니터 설정(액세스 모니터 설정 참조) 대화 상자에서 설정을 지정하십시오. 도어를 선택했으면 기본 설정을 유지하거나 필요한 경우 설정을 변경할 수 있습니다.
  4. 확인 을 클릭하면 액세스 모니터가 뷰에 추가됩니다.

    액세스 제어 사건이 발생하여 이벤트감시 시스템에서 발생하는 사전 정의된 인시던트로서 감시 시스템에서 동작을 트리거하는 데 사용됩니다. 감시 시스템 구성에 따라 이벤트는 외부 센서의 입력, 감지된 모션, 다른 응용 프로그램에 의해 발생되거나 사용자 입력을 통해 수동으로 발생시킬 수 있습니다. 예를 들어, 이벤트를 발생시켜 카메라 녹화를 특정 프레임 속도로 만들거나, 출력을 활성화하거나, 전자메일을 보내거나 이들을 조합하는 데 사용할 수 있습니다.를 트리거할 때, 항목 보기의 오른쪽에 그 내용이 표시됩니다.

액세스 모니터 설정

액세스 모니터에 대해 다음 설정을 지정합니다:

이름

설명
도어

액세스 제어 이벤트를 보려는 도어를 선택합니다. 도어를 선택하면 대화 상자의 나머지 설정이 현재 값으로 나타납니다.
소스

이벤트를 수신하려는 액세스 제어 소스의 유형을 선택합니다. 이 목록에는 예를 들어 도어 또는 도어에 대한 특정 액세스 지점이 포함될 수 있습니다. 액세스 지점은 카드 리더, 키패드, 센서 또는 버튼 등 관련 물리적 장치를 포함한 진입 지점입니다. 도어는 일반적으로 각각 도어를 통한 진입과 진출을 제어하는 두 개의 액세스 지점을 가지고 있습니다.

소스 목록은 시스템 관리자가 구성합니다.
카메라

이 도어에 관련된 비디오를 표시하려는 출처 카메라를 선택합니다. 기본적으로, 시스템은 시스템 관리자가 선택 도어와 연결한 카메라를 나열하지만 시스템의 다른 카메라를 선택할 수도 있습니다.
이벤트

받기 원하는 이벤트감시 시스템에서 발생하는 사전 정의된 인시던트로서 감시 시스템에서 동작을 트리거하는 데 사용됩니다. 감시 시스템 구성에 따라 이벤트는 외부 센서의 입력, 감지된 모션, 다른 응용 프로그램에 의해 발생되거나 사용자 입력을 통해 수동으로 발생시킬 수 있습니다. 예를 들어, 이벤트를 발생시켜 카메라 녹화를 특정 프레임 속도로 만들거나, 출력을 활성화하거나, 전자메일을 보내거나 이들을 조합하는 데 사용할 수 있습니다.의 유형을 선택합니다. XProtect 시스템 관리자가 정의한 이벤트 카테고리에서, 또는 액세스 제어 시스템에 정의된 이벤트 목록에서 이벤트를 선택할 수 있습니다.
명령

예를 들어, 도어 잠금 및 잠금 해제 등 액세스 모니터에서 사용하려는 명령 버튼을 선택합니다. 명령 목록의 내용은 해당 시스템 구성에 따라 결정됩니다.
순서

새 이벤트를 이벤트 목록의 상단이나 하단에 표시되게 하려는 경우 선택합니다.

액세스 모니터 설정 수정

라이브 탭에서 액세스 모니터의 설정을 변경할 수 있습니다.

  1. 설정 을 클릭하고 수정하려는 뷰 항목을 선택합니다.
  2. 속성 창에서 액세스 모니터 설정 버튼을 클릭합니다.
  3. 화면에 표시되는 액세스 모니터 설정(액세스 모니터 설정 참조) 대화 상자에서 설정을 지정하십시오.
  4. 확인 을 클릭하여 대화 상자를 닫은 다음 설정 을 클릭하여 실시간 보기로 돌아갑니다.

뷰 사용자 정의

오버레이 버튼라이브 탭 뷰에서 개별 카메라 위치에 마우스 커서를 이동할 때 비디오 위에 하나의 층으로 표시되는 버튼입니다. 오버레이 버튼을 사용하면 카메라에서 스피커, 이벤트, 출력을 작동시키고 PTZ 카메라를 이동하며 녹화를 시작하고 신호를 지울 수 있습니다.으로 인터페이스를 사용자 정의할 수 있습니다. 도어나 액세스 지점에 대해 구성된 명령 목록에서 뷰 항목에 액세스 제어용 오버레이 명령 버튼을 추가할 수 있습니다.

사용 예:

  • 액세스 모니터 외에 항목 보기의 명령 버튼에 직접 액세스합니다
  • 항목 보기에서 명령 버튼을 도어 바로 옆에 배치합니다
  • 액세스 모니터 설정에 특정된 것과는 다른 다른 명령 버튼을 추가하십시오

단계:

  1. 라이브 탭에서 설정 을 클릭하고 수정할 뷰 항목을 선택합니다.
  2. 오버레이 버튼 창에서 액세스 제어를 클릭합니다.
  3. 추가할 명령을 찾아 뷰 항목으로 끌어서 놓습니다.
  4. 설정 을 클릭하여 실시간 보기로 돌아갑니다.

    마우스를 뷰 항목 위로 끌어서 놓으면 오버레이 버튼이 나타납니다.

카드 소유자 정보 관리

액세스 제어 시스템에서 적합하게 설정이 되어 있다면 카드 소유자 레코드의 웹 페이지 표시로 직접 이동한 후, 예를 들어 사용자 관리를 수행하거나 카드 소유자에 관한 세부 정보를 가져올 수 있습니다.

플러그 인이 깊은 링크를 지원한다면 액세스 제어 시스템에 대해 다음 전제 조건이 존재합니다.

  • 반드시 웹 클라이언트를 포함
  • 반드시 딥 링크를 지원

카드 소유자 정보를 관리하려면:

  1. 액세스 제어 탭에서 카드 소유자 목록을 선택합니다.
  2. 카드 소유자를 검색하고 목록에서 해당자를 선택합니다.
  3. 오른쪽의 카드 소유자 정보 아래에서, 예를 들어 웹 페이지에 대한 링크를 클릭할 수 있습니다. 플러그 인에 따라 더 많은 링크가 지원될 수도 있고 추가 로그인 자격 증명을 요구할 수도 있습니다.
  4. 카드 소유자 정보와 액세스 권한 등 여러 가지 기능을 편집할 수 있습니다.
  5. 이 예에서는, 웹 페이지를 닫고 XProtect Smart Client (으)로 돌아갑니다.

액세스 요청 알림 설정 또는 해제

예를 들어, 한 사람만 액세스 요청을 처리해야 하는 경우에 액세스 요청 처리를 끌 수 있습니다.

  1. 을(를) 클릭한 후, 설정 을 클릭하여 설정 창을 엽니다.
  2. 액세스 제어 를 선택하고 액세스 요청 알림을 해제합니다.

나중에 액세스 요청을 다시 처리해야 하는 경우, 액세스 요청 알림을 켭니다. 액세스 요청 알림 내에서 설정 아이콘을 클릭하여 액세스 제어를 위한 옵션을 변경할 수도 있습니다.

서버 따라가기 필드를 선택하면 시스템 관리자가 액세스 요청 알림 표시 설정을 제어합니다.
