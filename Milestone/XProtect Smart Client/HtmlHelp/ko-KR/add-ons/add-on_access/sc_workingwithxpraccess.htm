XProtect Access (사용량)

하나 이상의 액세스 제어 시스템이 XProtect Access 추가 기능 제품을 통해 VMS"비디오 관리 소프트웨어"의 줄임말. 시스템과 통합된 경우, 도어를 모니터링하고, 도어 상태를 제어하며, 액세스 제어 이벤트를 조사하고 액세스 요청에 응답하며 카드 소유자 정보를 관리할 수 있습니다.

라이브 탭의 액세스 제어(설명됨)

라이브 탭에서는 비디오 오른쪽의 이벤트 목록과 함께 액세스 제어 소스와 연결된 카메라의 실시간 비디오를 볼 수 있습니다.

목록에서 이벤트 중 하나를 클릭하면 라이브 비디오가 자동으로 일시 정지되고 이벤트의 독립적 재생으로 전환됩니다. 라이브 비디오 보기로 돌아가려면 이벤트를 다시 클릭하거나 카메라 도구 모음에서 독립적 재생 아이콘을 클릭합니다(타임라인에 관계없이 레코딩된 비디오 보기 참조).

시스템 및 이벤트에 카드 소유자액세스 제어 시스템에서 인식할 수 있거나 또는 하나 이상의 영역, 건물 또는 유사한 곳에 액세스 권한을 부여하는 카드를 가진 사람. 액세스 제어도 참조하십시오. 정보가 있으면 선택된 이벤트의 카드 소유자 이름 옆에 있는 검색 아이콘을 클릭하여 액세스 제어 탭으로 이동한 후 이 사람과 관련된 모든 이벤트를 볼 수 있습니다.

맵을 통한 도어 모니터

맵 기능을 사용하여 감시와 액세스 제어 작업을 지원하는 경우, 액세스 제어 장치를 맵에 추가할 수 있습니다.

  1. 라이브 탭의 설정 모드에서 시스템 개요 창을 확장합니다.
  2. 목록에서 맵 을 선택하여 뷰 항목으로 끌어서 놓습니다.
  3. 맵 파일을 찾은 다음 확인 을 클릭합니다.
  4. 화면에 표시되는 맵 도구 상자에서 액세스 제어 추가를 클릭합니다.
  5. 나타나는 목록에서 도어 등의 관련 액세스 제어 장치를 맵으로 끌어서 놓습니다. 맵에 도어 아이콘이 나타납니다.
  6. 설정 을 클릭하여 실시간 보기로 변경합니다.
  7. 사람이 액세스를 요청하면 도어를 잠금 해제합니다. 도어가 잠금 해제되는 것은 누군가 액세스 요청 알림에 대해 명령 버튼을 사용해 또는 맵 상에서 출입을 허가했기 때문입니다. 액세스가 허가되면 도어 아이콘이 녹색으로 바뀌고 열린 문 형태로 나타납니다.
  8. 도어가 자동으로 또는 수동으로 다시 잠기면 도어 아이콘이 빨간색으로 바뀌고 닫힌 도어로 표시됩니다.
  9. 도어 아이콘을 마우스 오른쪽 버튼으로 클릭하고, 예를 들어 명령을 트리거할 수 있습니다.

    액세스 제어 장치의 상태는 항상 볼 수 있기 때문에, 이런 방식으로 사용되는 맵은 모니터링하고 있는 영역이나 건물에 대한 액세스 제어 장치 상태를 그래픽으로 빠르게 파악할 수 있습니다.

액세스 제어 이벤트 조사

액세스 제어 이벤트 검색 및 필터링

이벤트 목록을 필터링하여 원하는 데이터만 표시하는 몇 가지 방법이 있습니다.

  1. 액세스 제어 탭에서 이벤트 목록을 선택합니다.
  2. 목록 상단에서 필요한 필터를 클릭하고 기준을 지정합니다.
  3. 또는 목록 내의 특정 시간, 이벤트, 소스 또는 카드 소유자액세스 제어 시스템에서 인식할 수 있거나 또는 하나 이상의 영역, 건물 또는 유사한 곳에 액세스 권한을 부여하는 카드를 가진 사람. 액세스 제어도 참조하십시오.를 마우스 오른쪽 버튼으로 클릭하고 이 값을 이용하여 필터링할 수 있습니다.

    적용하는 필터는 목록에 즉시 반영됩니다.

다음에 대해 필터링을 적용할 수 있습니다.

이벤트 목록

설명
시간

사용 가능한 기간 중 하나를 선택하여 해당하는 특정 기간에 대한 데이터를 표시합니다. 예를 들어, 오늘 을 클릭하여 오늘 발생한 이벤트만 표시하거나 사용자 정의 간격을 사용하여 특정 기간을 지정합니다.

라이브 업데이트 를 선택하면 필터 기준에 맞는 새 이벤트가 발생할 경우 이벤트 목록이 즉시 업데이트됩니다. 목록에는 최대 100개의 이벤트가 표시됩니다. 라이브 업데이트 모드에서 작업할 때는 카드 소유자를 검색할 수 없습니다.
이벤트

이벤트 카테고리 및 분류되지 않은 이벤트 목록에서 사용 가능한 이벤트 유형을 하나 이상 직접 선택하거나 특정 액세스 제어 이벤트 중에서 선택합니다.
소스

도어 목록에서 사용 가능한 소스를 하나 이상 직접 선택하거나 다른 소스(예: 액세스 제어 시스템의 액세스 포인트 또는 컨트롤러) 중에서 선택하여 해당 장치에 대한 이벤트만 표시합니다.
액세스 제어 시스템

사용 중인 XProtect 시스템에 다수의 액세스 제어 시스템이 통합된 경우, 이벤트를 보려는 액세스 제어 시스템을 선택합니다.
카드 소유자

사용 가능한 카드 소유자를 하나 이상 선택합니다.

이벤트 목록(설명됨)

액세스 제어 탭에서 하나의 이벤트를 선택할 때 오른쪽의 미리보기에서 이 이벤트에 대한 관련 비디오 시퀀스를 볼 수 있습니다. 미리보기 카메라 제목 표시줄에 이벤트를 트리거한 장치에 관련된 카메라가 표시됩니다.

  • 하나의 도어에 여러 카메라가 연결되어 있는 경우, 카메라의 비디오가 미리보기에 모두 표시됩니다
  • 표준 재생 옵션은 도구 모음에서 사용할 수 있습니다
  • 선택한 이벤트에 관한 자세한 내용과 함께 비디오 미리보기 아래에 관련된 카드 소유자 정보가 표시됩니다
  • 을(를) 클릭하여 라이브 비디오를 보거나 레코딩된 비디오를 부동 창에 재생합니다

액세스 보고서 내보내기

라이브 업데이트 모드가 아닌 경우, 액세스 제어 탭에서 이벤트 목록의 보고서를 생성하여 PDF 파일로 내보낼 수 있습니다.

  1. 보고서에서 필요한 이벤트를 필터링하거나 검색합니다.

    이벤트 카운트가 매우 높을 경우, 검색 범위를 좁혀서 검색 결과를 줄일 것을 권장하는 메시지가 나타납니다.

  2. 액세스 보고서 버튼을 클릭합니다.
  3. 필드에 내용을 입력합니다. 보고서에 포함되는 내용은 다음과 같습니다.
    • 보고서 이름
    • 내보내기 대상 위치
    • 적용된 필터 목록
    • 주석 필드
    • 스냅샷을 포함하기 위한 옵션
  4. 확인 을 클릭하고 보고서가 완성될 때까지 기다립니다.
  5. 오른쪽위에서 세부 정보 를 클릭하고 나타나는 대화 상자에서 열기 를 클릭합니다.

    보고서가 PDF 형식으로 열립니다.

이벤트 목록의 라이브 업데이트 모드로 들어가고 나가기

라이브 탭에서 액세스 제어 이벤트의 라이브 비디오를 보는 대신, 액세스 제어 탭에서 라이브 업데이트 모드로 작업합니다. 이벤트 목록은 필터 기준에 맞는 새 이벤트가 발생할 경우 즉시 업데이트됩니다.

  1. 액세스 제어 탭에서 이벤트 목록을 선택합니다.
  2. 일반적으로 기간을 선택하는 필터의 드롭다운 목록에서 라이브 업데이트를 선택합니다.

    검색 필드 옆에서 모드가 변경되었고 필터 기준을 충족하는 이벤트가 발생하는 경우 즉시 목록이 업데이트되는 것을 확인할 수 있습니다.

    라이브 업데이트 모드에서 작업할 때는 카드 소유자를 검색할 수 없으며 액세스 보고서를 생성할 수 없습니다.

  3. 라이브 업데이트 모드에서 나가려면 새 기간을 필터링합니다.

도어 상태 모니터 및 제어

도어 목록에는 각 액세스 제어 시스템에 있는 도어, 액세스 지점 및 기타 액세스 제어 장치의 목록과 이들의 현재 상태가 나열됩니다. 예를 들어, 특정 도어의 상태를 알아야 하는 경우 유용합니다.

도어 목록을 필터링하여 원하는 데이터만 표시하는 몇 가지 방법이 있습니다.

  1. 액세스 제어 탭에서 도어 목록을 선택합니다.
  2. 목록 상단에서 필요한 필터를 클릭하고 기준을 지정합니다.
  3. 필터를 결합하거나 검색 필드에 기준을 입력하여 도어를 검색할 수 있습니다.
  4. 또는 목록 내의 도어 또는 상태를 마우스 오른쪽 버튼으로 클릭하고 이 값을 이용하여 필터링할 수 있습니다.

    적용하는 필터는 목록에 즉시 반영됩니다.

필터링할 수 있는 사항은?

도어 목록

설명
이름

사용 가능한 도어, 액세스 포인트 및 분류되지 않은 유형을 하나 이상 선택하거나 다른 액세스 제어 장치 중에서 선택하여 선택 항목의 상태만 표시합니다.
액세스 제어 시스템

사용 중인 XProtect 시스템에 다수의 액세스 제어 시스템이 통합된 경우, 도어를 보려는 액세스 제어 시스템을 선택합니다.
상태

상태 카테고리 및 분류되지 않은 상태 목록에서 사용 가능한 상태를 하나 이상 직접 선택하거나 특정 액세스 제어 상태 중에서 선택합니다.

감시 구역에 대해 문의 상태를 모니터하는 다른 방법은 문을 맵에 추가하는 것입니다(맵을 통한 도어 모니터 참조).

도어 목록(설명됨)

액세스 제어 탭에서 도어 목록에 있는 하나의 도어를 선택할 경우, 그와 연결된 카메라가 화면의 오른쪽에 상세 정보와 함께 라이브 비디오를 표시합니다.

  • 하나의 도어에 여러 카메라가 연결되어 있는 경우, 카메라의 비디오가 미리보기에 모두 표시됩니다
  • 도구 모음에서 표준 독립적 재생 옵션을 사용할 수 있습니다
  • 동작 버튼을 사용하여 도어 잠금/잠금 해제와 같이 해당 도어와 관련된 특정 명령을 실행할 수 있습니다. 사용 가능한 명령은 시스템 구성에 따라 좌우됩니다
  • 선택된 도어와 관련 있는 정보가 라이브 비디오 미리보기 아래에 표시됩니다
  • 을(를) 클릭하여 라이브 비디오를 보거나 레코딩된 비디오를 부동 창에 재생합니다

카드 소유자 조사

카드 소유자 목록에는 각 액세스 제어 시스템에 있는 카드 소유자 목록과 그 세부 정보가 나열됩니다. 예를 들어, 특정인에 대한 세부 정보가 필요한 경우 유용합니다.

카드 소유자 목록을 필터링하여 원하는 데이터만 표시하는 몇 가지 방법이 있습니다.

  1. 액세스 제어 탭에서 카드 소유자 목록을 선택합니다.
  2. 목록 상단에 있는 필터를 클릭하여 카드 소유자를 조사하려는 액세스 제어 시스템을 지정합니다. 한 번에 하나의 액세스 제어 시스템으로만 작업할 수 있습니다.
  3. 필터를 결합하거나 검색 필드에 기준을 입력하여 카드 소유자를 검색할 수 있습니다.
  4. 또는 목록 내의 카드 소유자 또는 유형을 마우스 오른쪽 버튼으로 클릭하고 이 값을 이용하여 필터링할 수 있습니다.

    적용하는 필터는 목록에 즉시 반영됩니다.

필터링할 수 있는 사항은?
카드 소유자 목록 설명
이름

사용 가능한 카드 소유자 중 한 명을 선택하여 이 사람에 대한 세부 정보를 표시합니다.
유형

사용 가능한 카드 소유자 유형 중 하나를 선택하여 이 유형의 카드 소유자 목록을 표시합니다.

카드 소유자를 선택하면 화면 오른쪽에 이 사람에 관한 세부 정보가 표시됩니다. 사용 중인 시스템에 따라 여기에는 사진 또는 액세스 제어 시스템의 카드 소지자 관리를 위한 링크가 포함될 수 있습니다(카드 소유자 정보 관리 참조).

액세스 요청 알림(설명됨)

사람이 건물에 들어가려고 할 때 보안 요원만 도어를 열어야 하는 것으로 해당 조직에서 정해 놓았을 수 있습니다. 이러한 조건이 적용되면 예를 들어, 어떤 사람이 하나 이상의 영역에 들어가려고 할 때 액세스 요청 알림을 받을 수 있습니다. 액세스 요청 알림을 트리거하는 모든 조건은 비디오 관리 시스템에서 지정해야 합니다. 이 알림에는 액세스 요청과 관련된 실시간 비디오가 표시되고, 액세스를 요청하는 사람을 볼 수 있습니다. 열어야 하는 도어 이름이 예를 들어, 액세스 요청 - 정문을 나타내는 제목으로 표시됩니다. 도어 상태(예: 열림, 닫힘 또는 강제 열림)도 표시됩니다. 하나의 도어에 여러 카메라가 연결되어 있는 경우, 그 모두가 각각의 아래에 표시됩니다.

액세스 요청 알림은 일시적입니다. 액세스 요청 알림을 닫으면 알림이 더 이상 시스템에 남아 있지 않습니다. 액세스 요청 알림이 표시되는 동안 XProtect Smart Client 을(를) 닫으면, 다시 시작할 때 알림이 복원되지 않습니다.

액세스 요청 알림 관리(설명됨)

XProtect Smart Client 이(가) 실행되고 있다면, 다른 응용 프로그램에서 작업하는 경우에 화면에 액세스 요청 알림 팝업이 표시됩니다.

부동 창에서 라이브 비디오를 보려면 을(를) 클릭합니다.

액세스 요청은 액세스 요청 알림 창에서 쌓여지며 표시되므로 동일한 알림 창에서 들어오는 모든 액세스 요청 알림을 처리할 수 있습니다. 알림을 화면의 다른 쪽, 심지어 다른 화면으로(연결되어 있는 경우) 끌어서 놓을 수 있습니다.

필요한 경우, 액세스 요청 알림 창을 최소화시켜 이 기능을 백그라운드로 계속 유지할 수 있습니다. 새 알림이 있으면 작업 표시줄에서 XProtect Smart Client 아이콘이 깜빡입니다.

액세스 요청에 대응

사용 중인 VMS"비디오 관리 소프트웨어"의 줄임말. 시스템이 양방향 오디오를 지원하고 스피커와 마이크가 액세스 요청 알림을 표시하는 관련 카메라에 부착되어 있으면 액세스 요청 알림 과정에서 진입을 원하는 사람에게 말하고 그의 말을 들을 수 있습니다.

  1. 액세스를 요청하는 사람이 하는 말을 들으려면 버튼을 클릭합니다.
  2. 예를 들어, 해당 영역에서 해야 할 조치를 알려주기 위해 액세스를 요청하는 사람에게 말하려면 버튼을 클릭한 상태로 유지합니다.
  3. 다른 동작을 실행하려면, 마이크로폰과 스피커 버튼 오른쪽의 명령 버튼을 이용하십시오. 가장 일반적인 동작은 액세스를 요청하는 사람을 위해 도어의 잠금을 해제하는 것이지만 관련 출입문 주변 영역의 조명을 켜는 동작이 될 수도 있습니다.

액세스 제어건물이나 영역에 개인, 차량 등의 출입을 제어하는 보안 시스템. 시스템이 XProtect 시스템에 카드 소유자 정보를 제공하는 경우, 이 정보를 이용할 수 있습니다. 카드 소유자 정보의 예: 카드 소유자의 ID 번호, 이름, 부서, 전화 번호 및 권한 수준. 시스템 구성에 따라 카드 소지자 정보 관리를 할 수도 있습니다(카드 소유자 정보 관리 참조).
