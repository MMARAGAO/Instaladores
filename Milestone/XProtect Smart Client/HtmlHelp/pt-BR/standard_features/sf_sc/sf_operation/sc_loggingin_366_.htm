Fazendo login

Login e saída

  1. Iniciar XProtect Smart Client. A janela de login aparece.

  2. Especifique o nome ou endereço do servidor ao qual você quer se conectar.
  3. Selecione um destes métodos de autenticação:

    • Autenticação Windows (usuário atual) - selecione esta opção para fazer o login com suas credenciais de usuário atuais do Windows

    • Autenticação Windows - selecione esta opção para fazer o login com credenciais de usuário do Windows, diferentes das credenciais de usuário Windows usadas no momento.

    • Autenticação básica - selecione esta opção para fazer o login como um usuário básico. Usuários básicos são definidos pelo administrador do sistema em XProtect Management Client

    • [Nome do external IDPUma entidade externa que pode ser associada ao VMS XProtect para gerenciar informações de identidade do usuário e fornecer serviços de autenticação de usuário ao VMS.] - selecione esta opção para fazer login com um external IDP. Consulte também Faça login por meio de um external IDP.

  4. Clique em Conectar. Se ocorrer um problema durante o login, você poderá receber uma mensagem de erro. Consulte também Efetuando login (solução de problemas).
  5. Dependendo da configuração, você pode ser solicitado a restaurar as visualizações usadas na sessão anterior:
    • Visualização principal - esta opção restaura a visualização usada na última vez na janela principal.
    • Visualizações separadas - esta opção restaura a visualização usada na última vez na janela flutuante. Somente disponível ao conectar a sistemas VMS XProtect. Consulte também Diferenças entre sistemas de vigilância
  6. Para sair do XProtect Smart Client, basta clicar no botão Log out na barra de título.

Se encontrar um segundo diálogo durante o login, você precisa de autorização de login adicional para obter acesso ao XProtect Smart Client.

Autorização de login (explicado)

Quando você efetuar login no XProtect Smart Client, a solicitação de autorização adicional de seu login poderá ser exibida. Você precisará de seu supervisor, administrador de sistema, ou alguém que tenha permissão de autorizar você a digitar suas credenciais, juntamente com a sua na janela de login. Depois disso, você poderá continuar.

Se não sabe quem pode autorizá-lo, pergunte ao administrador do sistema.

Fazendo login nos sistemas de controle de acesso (explicado)

Quando fizer login no XProtect Smart Client, pode ser solicitado que você insira logins adicionais nos sistemas de controle de acesso, se eles estiverem configurados para fazê-lo.

Seu login controla as partes de uma integração de controle de acesso, por exemplo, portas, que você pode gerenciar e operar.

Se você não souber suas credenciais de login para um sistema de controle de acesso, solicite ao administrador do sistema.

O sistema memoriza as suas credenciais de login, de forma que você só precisa preencher suas credenciais na primeira vez que efetuar login ou o login falhar.

Alterar senha no XProtect Smart Client

Se fizer o login como usuário básico (Autenticação básica), poderá alterar a sua senha. Se você escolher um método de autenticação diferentes, somente o administrador do sistema pode alterar a sua senha. A alteração frequente da sua senha aumenta a segurança do seu sistema VMS XProtect.

Requisitos

A versão do seu sistema VMS XProtect deve ser 2021 R1 ou posterior.

Etapas:

  1. Iniciar XProtect Smart Client. A janela de login é exibida.

  2. Especifique suas informações de login. Na lista Autenticação, selecione Autenticação básica. Um link com o texto Alterar senha aparece.

  3. Clique no link. Uma janela do navegador abre.
  4. Siga as instruções e salve suas alterações.
  5. Faça o login no XProtect Smart Client usando a sua nova senha.

Permitir conexões que usam um modelo de segurança mais antigo (HTTP)

Se o servidor VMS XProtect em que você está tentando fazer login não tem um certificado instalado, então você não pode se conectar com um protocolo de rede HTTPS, o mais novo modelo de segurança disponível em XProtect. Nesses casos, será solicitado que você permita conexões com um modelo de segurança mais antigo (HTTP).

Se selecionar a caixa de seleção Lembrar da minha escolha. Não exibir esta caixa de seleção de mensagem novamente, conexões HTTP sempre serão permitidas no futuro. Consulte também Limpar configuração que permite conexões que usam um modelo de segurança mais antigo.

Limpar configuração que permite conexões que usam um modelo de segurança mais antigo

Você pode limpar a configuração que permite fazer login em um servidor VMS XProtect usando um protocolo de rede com uma conexão que usa um modelo de segurança mais antigo (HTTP). Na próxima vez que você fizer o login, será solicitado que permita conexões HTTP.

A configuração se aplica somente à sua conta de usuário e à máquina na qual você está trabalhando no momento.

Requisitos

Durante o processo de login, você permitiu conexões HTTP e selecionou Lembrar minha escolha. Não mostrar esta mensagem novamente. Consulte também Permitir conexões que usam um modelo de segurança mais antigo (HTTP).

Etapas:

  1. No canto superior direito, clique no botão Status. Uma janela aparece.

  2. Selecione a guia Informações de login.

  3. Clique no botão Limpar.
  4. Clique em OK para fechar a janela. Na próxima vez que você tentar fazer o login, será solicitado que permita conexões HTTP.
